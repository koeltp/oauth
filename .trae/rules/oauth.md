# Trae 规则 - 必须严格遵守

1. **清理无用代码**  
   在每次代码生成或修改后，自动移除未使用的变量、导入、函数。若不确定是否无用，先询问用户。

2. **语言偏好**  
   所有回复、思考过程、Thought、代码注释、日志输出 **必须使用简体中文**。除非用户明确要求英文。

3. **修改前同步代码**  
   在编辑任何文件前获得最新的文件，万一在你编辑之前用户更新了。

4. **文件创建规范**  
   创建新文件时，先扫描项目中同类型文件的存放位置（如组件放 `src/components/`，工具函数放 `src/utils/`）。遵循项目现有的目录结构和命名约定。

5. **代码注释**  
   为函数、复杂逻辑、关键类型定义添加中文注释。注释要说明“为什么”而不是“做什么”。
